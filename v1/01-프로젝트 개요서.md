# 01. 프로젝트 개요서

> **프로젝트명**: 중고거래 플랫폼 (가칭: 마켓플레이스)
> **작성일**: 2025-06
> **작성자**: SA 설계팀
> **버전**: v1.0

---

## 1. 프로젝트 목적

당근마켓 스타일의 중고거래 플랫폼 백엔드 시스템을 설계·구현한다.
단순 CRUD를 넘어, **동시성 제어**, **분산 캐싱**, **실시간 채팅** 등 실무에서 요구되는 기술을 직접 적용하는 것이 핵심 목표이다.

---

## 2. 배경 및 문제 정의

| 구분 | 내용 |
|------|------|
| 학습 배경 | 경력 3개월 백엔드 개발자 4명이 실무 수준의 아키텍처를 경험 |
| 해결할 기술 문제 | 여러 사용자가 동시에 같은 상품을 구매할 때 데이터 정합성 보장 |
| 해결할 기술 문제 | 반복 조회되는 인기 검색어에 대한 DB 부하 감소 |
| 해결할 기술 문제 | 구매자-판매자 간 실시간 채팅 |
| 비즈니스 목표 | 3주 안에 동작하는 프로덕션급 백엔드 완성 |

---

## 3. 프로젝트 범위

### In Scope

- 회원 가입 / 로그인 / JWT 인증
- 상품 등록 / 수정 / 삭제 / 조회
- 상품 검색 (키워드 + 카테고리 필터)
- 거래 상태 관리 (판매중 → 예약중 → 거래중 → 판매완료 → 리뷰)
- 1:1 채팅 (구매자-판매자, WebSocket/STOMP)
- 관심 상품 (찜하기)
- 동시성 제어 (상품 구매 시 1명만 예약 가능)
- 인기 검색어 캐싱 (Caffeine 로컬 캐시)
- Redis 분산락 (Redisson, 도전 기능)
- Redis 캐싱 Cache-Aside 패턴 (도전 기능)
- EXPLAIN 기반 DB 인덱스 최적화 (도전 기능)
- AWS 인프라 배포 자동화 (도전 기능)

### Out of Scope

- 결제 연동 (PG사 연동 없음 — 거래 완료는 상태 변경으로 처리)
- 신고 / 제재 시스템
- 알림 (Push / FCM)
- 관리자 페이지
- 지도 기반 위치 검색

## 4. 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|-----------|
| Language | Java 17 | LTS, Record/Pattern matching 등 최신 문법 활용 |
| Framework | Spring Boot 3.x | 업계 표준, Auto-configuration |
| ORM | Spring Data JPA + QueryDSL | 기본 CRUD는 JPA, 복잡한 동적 쿼리는 QueryDSL |
| DB | MySQL 8 | 관계형 데이터, JSON 컬럼 지원, EXPLAIN 최적화 |
| Cache (Local) | Caffeine | 인기 검색어 등 단일 인스턴스 로컬 캐시 |
| Cache (분산) | Redis (ElastiCache) | 세션 공유, 분산락, Cache-Aside |
| 분산락 | Redisson | Redis 기반 분산락 구현체 표준 |
| 실시간 통신 | WebSocket + STOMP | 채팅 메시지 실시간 송수신 |
| 컨테이너 | Docker | 환경 일관성, ECR 배포 |
| CI/CD | GitHub Actions | 코드 푸시 → 빌드 → ECR → EC2 자동 배포 |
| 클라우드 | AWS (EC2, RDS, ElastiCache, ECR) | 업계 표준 인프라 |

## 7. 핵심 제약 사항

1. **3주 내 완료** — 도전 기능은 필수 기능 완료 후 추가
2. **경력 3개월 팀** — 코드 복잡도보다 명확성 우선
3. **결제 없음** — 거래 완료는 판매자의 상태 변경으로 처리
4. **단일 서버 기준 설계** — 스케일아웃은 고려하되 초기 배포는 단일 EC2
