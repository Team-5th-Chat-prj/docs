## 3. EC2 Docker Compose 구성

### HTTPS 처리 주체: Nginx Reverse Proxy

ALB 없이 단일 EC2에서 HTTPS를 처리하는 방법은 두 가지다.

| 방법 | 장점 | 단점 | 선택 |
|------|------|------|------|
| Nginx + Let's Encrypt | 무료, EC2 내 완결 | 인증서 갱신 관리 필요 | ✅ 채택 |
| ALB (AWS Certificate Manager) | 관리 편의, 자동 갱신 | 월 ~$18 추가 비용 | 도전 기능 이후 고려 |

> **현재 전략**: EC2 내부에 Nginx 컨테이너를 추가하여 80/443 → Spring Boot 8080으로 프록시한다.
> WebSocket(STOMP)은 `wss://` 연결이 필요하므로 TLS 없이는 배포 환경에서 동작하지 않는다.

```yaml
# docker-compose.yml (EC2에서 실행)
version: '3.8'
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/certs:/etc/nginx/certs:ro   # Let's Encrypt 인증서
    depends_on:
      - app
    restart: unless-stopped

  app:
    image: {AWS_ACCOUNT_ID}.dkr.ecr.ap-northeast-2.amazonaws.com/marketplace:latest
    expose:
      - "8080"          # 외부에 직접 노출하지 않음 — Nginx를 통해서만 접근
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - DB_URL=${DB_URL}
      - DB_USERNAME=${DB_USERNAME}
      - DB_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=${REDIS_HOST}
      - JWT_SECRET=${JWT_SECRET}
    restart: unless-stopped
    healthcheck:
      # Spring Boot Actuator health 엔드포인트
      # 의존성: spring-boot-starter-actuator
      # 응답: {"status":"UP"} → 200 OK
      # 실패 시: Docker가 컨테이너를 unhealthy로 표시 → Nginx가 해당 upstream 제외
      test: ["CMD", "curl", "-f", "http://localhost:8080/actuator/health"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 40s   # 앱 구동 완료 대기
```

**Nginx 설정 핵심 (`nginx/nginx.conf`)**:
```nginx
server {
    listen 443 ssl;
    server_name api.marketplace.com;

    ssl_certificate     /etc/nginx/certs/fullchain.pem;
    ssl_certificate_key /etc/nginx/certs/privkey.pem;

    # WebSocket 업그레이드 헤더 (STOMP 필수)
    location /ws-chat {
        proxy_pass http://app:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    location / {
        proxy_pass http://app:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# HTTP → HTTPS 리다이렉트
server {
    listen 80;
    return 301 https://$host$request_uri;
}
```

## 4. GitHub Actions Workflow 구조

```yaml
# .github/workflows/deploy.yml
name: Deploy to AWS EC2

on:
  push:
    branches: [ main ]

jobs:
  # ── 1단계: 테스트 ──────────────────────────────────────────────
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Run Tests
        # -x test 제거: CI에서 테스트 스킵은 main 브랜치 품질 보장 불가
        # H2 인메모리 DB로 슬라이스 테스트 실행 (application-test.yml 필요)
        run: ./gradlew test

  # ── 2단계: 빌드 & 배포 ─────────────────────────────────────────
  build-and-deploy:
    needs: test        # 테스트 통과 후에만 배포 진행
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'

      - name: Build with Gradle (테스트 스킵 — 위 job에서 이미 통과)
        run: ./gradlew build -x test

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v2
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ap-northeast-2

      - name: Login to Amazon ECR
        uses: aws-actions/amazon-ecr-login@v1

      - name: Build and Push Docker image
        run: |
          docker build -t $ECR_REGISTRY/$ECR_REPOSITORY:latest .
          docker push $ECR_REGISTRY/$ECR_REPOSITORY:latest

      - name: Deploy to EC2
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ec2-user
          key: ${{ secrets.EC2_SSH_KEY }}
          script: |
            aws ecr get-login-password --region ap-northeast-2 | \
              docker login --username AWS --password-stdin $ECR_REGISTRY
            docker pull $ECR_REGISTRY/$ECR_REPOSITORY:latest
            docker-compose down
            docker-compose up -d
            sleep 50
            curl -f http://localhost:8080/actuator/health || exit 1
```

> **ALB 미적용 이유**: 학습 목적 3주 프로젝트에서 ALB($18/월)는 불필요한 비용. Nginx로 TLS + 리버스 프록시를 직접 구성하여 인프라 전체 흐름을 학습한다. 실무 전환 시 ALB + ACM으로 교체.
> **Blue/Green 미적용 이유**: 단일 EC2 환경에서 Blue/Green은 서버 2대가 필요. 3주 프로젝트 허용 다운타임 내 `docker-compose down → up`으로 충분.
