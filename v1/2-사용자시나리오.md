## 2. 사용자 시나리오

### 시나리오 1. 판매자가 상품을 등록하고 거래를 완료한다

**페르소나**: 김민준 (판매자)

```
1. 앱에 로그인한다 (JWT 발급)
2. [상품 등록] 버튼을 누른다
3. 카테고리 선택: 가구/침대 프레임
4. 제목, 설명, 가격, 사진을 입력하고 등록한다
   → 상태: "판매중"
5. 이서연에게 채팅이 온다: "아직 판매 중인가요?"
6. 민준은 채팅으로 답장하고 예약을 확정한다
   → 상태: "예약중" (이 시점부터 다른 사람은 예약 불가)
7. 이서연을 만나 물건을 전달한다
   → 상태: "거래중" → "판매완료"
8. 이서연이 리뷰를 작성한다
   → 상태: "리뷰완료"
```

**핵심 기술 포인트**: 6번 단계에서 동시에 여러 사용자가 예약을 시도할 경우, 비관적 락 + Redisson 분산락으로 정확히 1명만 예약 성공

---

### 시나리오 2. 구매자가 상품을 검색하고 예약한다

**페르소나**: 이서연 (구매자)

```
1. 앱을 실행한다 (비로그인 상태에서도 검색 가능)
2. 검색창에 "침대 프레임"을 입력한다
   → Caffeine 캐시에서 인기 검색어 자동완성 제공
3. 카테고리 필터: "가구 > 침대/매트리스"로 좁힌다
4. 상품 목록에서 민준의 상품을 클릭한다
5. 상세 페이지에서 사진과 설명을 확인한다
6. [채팅하기] 버튼을 누른다 → 로그인 유도 (미로그인 시)
7. 로그인 후 채팅방이 생성된다
8. "아직 판매 중인가요?" 메시지를 보낸다
9. 민준의 응답을 받고 [예약하기] 요청을 보낸다
   → 판매자가 수락하면 상태: "예약중"
```

**핵심 기술 포인트**: 2번 인기 검색어는 Caffeine TTL 10분 캐시, 3번 카테고리 필터는 QueryDSL 동적 쿼리

---

### 시나리오 3. 동시에 두 명이 같은 상품을 예약 시도한다

**페르소나**: 이서연 + 또 다른 구매자 최우진

```
1. 이서연과 최우진이 거의 동시에 같은 상품 상세 페이지를 보고 있다
2. 두 사람이 동시에 [예약하기] 버튼을 클릭한다
3. 서버에서 Redisson 분산락 획득 시도
   → 이서연이 0.001초 먼저 락 획득
   → 최우진은 락 대기 → 타임아웃 → 실패 응답
4. 이서연: 예약 성공 (상태 "예약중")
5. 최우진: "이미 예약된 상품입니다" 응답 수신
   → 찜하기만 가능, 채팅은 가능
```

**핵심 기술 포인트**: Redisson tryLock(waitTime=0, leaseTime=3s) — 대기 없이 즉시 실패 반환으로 사용자 경험 개선

---

### 시나리오 4. 재판매자가 여러 상품을 관리한다

**페르소나**: 박준서 (재판매자)

```
1. 로그인 후 [내 판매 목록] 진입
2. 등록된 상품 5개 목록 확인 (상태별 필터: 판매중/예약중/완료)
3. "판매중" 상품 3개 확인
4. 특정 상품의 가격을 수정한다
5. 채팅 알림이 온 상품 2개 확인
6. 각 채팅방에서 구매자와 협의 진행
7. 구매자 확정 후 상태를 "예약중"으로 변경
```

## 3. 사용자 여정 요약

```
[비회원]
  └─ 상품 목록 조회 (가능)
  └─ 상품 상세 조회 (가능)
  └─ 검색 (가능)
  └─ 채팅 시도 → 로그인 유도

[회원 - 구매자]
  └─ 로그인/회원가입
  └─ 검색 + 필터
  └─ 상품 상세 조회
  └─ 찜하기
  └─ 채팅 (1:1)
  └─ 예약 요청
  └─ 거래 완료 확인
  └─ 리뷰 작성

[회원 - 판매자]
  └─ 상품 등록/수정/삭제
  └─ 채팅 응답
  └─ 예약 수락/거절
  └─ 거래 상태 변경 (예약중 → 거래중 → 판매완료)
  └─ 리뷰 확인
```
