# 01. 프로젝트 개요서

> **프로젝트명**: 중고거래 플랫폼 (명칭: 겟츄)
> **작성일**: 2026-04-09 최초 / 2026-04-23 최신
> **작성자**: SA 설계팀
> **버전**: v1.4

---

## 1. 프로젝트 목적

당근마켓 스타일의 중고거래 플랫폼 백엔드 시스템을 설계·구현한다.
단순 CRUD를 넘어, **동시성 제어**, **분산 캐싱**, **실시간 채팅**, **위치 기반 서비스** 등 실무에서 요구되는 기술을 직접 적용하는 것이 핵심 목표이다.

## 3. 프로젝트 범위

### In Scope

- 회원 가입 / 로그인 / 로그아웃 / JWT 인증
- 상품 등록 / 수정 / 삭제 / 조회
- 상품 검색 (키워드 + 카테고리 필터, QueryDSL)
- 거래 상태 관리 (판매중 → 예약중 → 거래중 → 판매완료 → 리뷰)
- 1:1 채팅 (구매자-판매자, WebSocket/STOMP)
- 관심 상품 (찜하기)
- 동시성 제어 (상품 구매 시 1명만 예약 가능)
- 인기 검색어 캐싱 (Caffeine 로컬 캐시)
- **위치 기반 동네 인증** (카카오 지도 API — GPS 좌표 / 주소 텍스트 입력)
- **근처 상품 조회** (인증된 동네 기준 반경 내 상품 거리 오름차순 제공)
- 프론트엔드 구성 (별도 레포지토리로 관리, 백엔드 API 연동 확인용)
- Redis 분산락 (Lettuce SETNX 직접 구현, 도전 기능)
- Redis 캐싱 Cache-Aside 패턴 (도전 기능)
- EXPLAIN 기반 DB 인덱스 최적화 (도전 기능)

### Out of Scope

- 관리자 페이지

## 4. 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|-----------|
| Language | Java 17 | LTS, Record/Pattern matching 등 최신 문법 활용 |
| Framework | Spring Boot 3.x | 업계 표준, Auto-configuration |
| ORM | Spring Data JPA + QueryDSL | 기본 CRUD는 JPA, 복잡한 동적 쿼리는 QueryDSL |
| DB | MySQL 8 | 관계형 데이터, EXPLAIN 최적화, Spatial(GEOMETRY/POINT) 지원 |
| Spatial | Hibernate Spatial + JTS | MySQL GEOMETRY 타입 매핑, Java 공간 데이터 객체 생성·변환 |
| 외부 API | 카카오 지도 API | 주소→좌표 변환(Geocoding), 좌표→행정구역명 변환(Reverse Geocoding) |
| Cache (Local) | Caffeine | 인기 검색어 등 단일 인스턴스 로컬 캐시 |
| Cache (분산) | Redis + Lettuce | Cache-Aside, SETNX 기반 분산락 직접 구현 |
| 실시간 통신 | WebSocket + STOMP | 채팅 메시지 실시간 송수신 |
| 문서화 | Spring REST Docs + restdocs-api-spec | 테스트 기반 API 문서 자동 생성 및 OpenAPI YAML 추출 |
| 컨테이너 | Docker | 환경 일관성, ECR 배포 |

## 7. 핵심 제약 사항

1. **3주 내 완료** — 도전 기능은 필수 기능 완료 후 추가
2. **경력 3개월 팀** — 코드 복잡도보다 명확성 우선
3. **결제 없음** — 거래 완료는 판매자의 상태 변경으로 처리
4. **단일 서버 기준 설계** — 스케일아웃은 고려하되 초기 배포는 단일 EC2
